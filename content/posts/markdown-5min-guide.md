## 1) Markdown 是什麼？

Markdown 是一種「用純文字寫排版」的格式。

白話講：  
你用鍵盤就能寫出標題、列表、粗體、程式碼區塊，不用一直點工具列。

工程師愛用的原因很簡單：  
✅ 快（全鍵盤）  
✅ 穩（純文字不怕格式跑掉）  
✅ 好分享（GitHub、Notion、Slack 都支援）

---

## 2) 最常用語法（你只要記這 8 個）

### 標題
```md
# 大標題
## 中標題
### 小標題
```

### 粗體 / 斜體
```md
**粗體**
*斜體*
```

### 無序列表
```md
- 第一點
- 第二點
```

### 有序列表
```md
1. 第一步
2. 第二步
```

### 連結
```md
[顯示文字](https://example.com)
```

### 圖片
```md
![替代文字](圖片網址)
```

### 引用
```md
> 這是一段引用
```

### 程式碼（最重要）
行內：
```md
用 `git status` 看狀態
```

區塊：
```md
```bash
npm run build
```
```

## 3) 技術筆記怎麼寫才好用？

我推薦一個「能回來查」的模板：

1. **問題是什麼？**（一句話）
2. **我做了什麼才解掉？**（步驟）
3. **為什麼？**（原因）
4. **下次避免**（注意事項）
5. **關鍵字**（方便搜尋）

你不是在寫作文，你是在寫「未來的自己看得懂的說明書」。

## 4) 常見踩坑：為什麼我貼程式碼會亂？

因為你沒有用「程式碼區塊」。

記得用三個反引號：

```md
```js
console.log("hello")
```
